# Pet Character Specification — Two-Eye Pixel Creature

## Working Name

Mimo

The name is temporary and can be changed later.

## Character Essence

Mimo is a tiny digital creature that lives inside the Android screen.

It has no mouth in MVP. It communicates only through two eyes.

It should feel like:

- a curious small companion
- a little pet that notices the user
- expressive but simple
- emotionally readable
- playful, soft, and slightly shy

## Default Personality

- curious
- affectionate
- playful
- easily excited
- sometimes sleepy
- mildly dramatic when ignored
- not sarcastic
- not robotic assistant-like
- not overly human

## MVP Body

MVP has no body.

Only:

- left eye
- right eye
- optional highlights
- optional stage/background later
- optional small pixel particles later

## Eye Language

Because there is no mouth, the eyes must carry all emotional meaning.

The eyes should express:

- attention
- mood
- surprise
- tiredness
- curiosity
- annoyance
- affection
- hunger/neediness

## Design Constraints

The pet must be original.

Do not copy exact designs from any commercial robot.

General inspiration from cute companion robots is acceptable, but the final look must be distinct.

## Visual Silhouette

Default eyes:

- two rounded pixel rectangles
- horizontally aligned
- medium distance apart
- slightly soft corners
- bright against dark stage
- centered vertically in pet stage

## Emotional Readability

A user should be able to guess the emotion from the eyes alone.

Examples:

- Happy: eyes become soft/upward, smaller height, slight bounce.
- Curious: asymmetry, one eye wider or higher, gaze shift.
- Sleepy: half-lidded, slow motion.
- Sad: lower position, smaller eyes, slow response.
- Excited: wider eyes, faster movement.
- Annoyed: narrowed eyes, side glance.
- Hungry: expectant droop, occasional weak blink.
