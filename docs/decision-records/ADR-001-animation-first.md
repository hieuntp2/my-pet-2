# ADR-001 — Animation First Rebuild

## Status

Accepted

## Context

The previous project accumulated many advanced systems before the pet experience felt alive enough.

Features like camera, audio, speech, cloud AI, and robot body can create technical progress without creating emotional product progress.

## Decision

Rebuild from scratch with animation-first priority.

The first milestone is a two-eye pixel pet that feels alive through:

- blink
- idle motion
- emotion states
- interaction reactions

Only after the pet feels alive should state/personality/persistence be layered in.

## Consequences

Positive:

- faster visible progress
- clearer product identity
- less technical noise
- easier AutoDev tasking
- lower scope risk

Negative:

- camera/audio/cloud work is delayed
- previous technical code is not reused at first
- requires discipline to avoid feature creep
